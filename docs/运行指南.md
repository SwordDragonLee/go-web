# 项目运行指南

## 前置条件

1. **Go 环境**：Go 1.21 或更高版本

   ```bash
   go version
   ```

2. **MySQL 数据库**：MySQL 5.7+ 或 MariaDB
   - 确保 MySQL 服务正在运行
   - 创建数据库（如果不存在）

## 运行步骤

### 1. 创建数据库

登录 MySQL 并创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS go_web CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 配置数据库连接

编辑 `configs/config.yaml` 文件，修改数据库配置：

```yaml
database:
  host: localhost # 数据库主机地址
  port: 3306 # MySQL端口
  user: root # 数据库用户名
  password: your_password # 数据库密码
  database: go_web # 数据库名
```

或者使用 DSN 方式：

```yaml
database:
  dsn: "root:your_password@tcp(localhost:3306)/go_web?charset=utf8mb4&parseTime=True&loc=Local"
```

### 3. 安装依赖

```bash
go mod download
```

或者

```bash
go mod tidy
```

### 4. 运行项目

#### 方式一：直接运行（开发环境）

```bash
go run cmd/api/main.go
```

#### 方式二：编译后运行（生产环境）

```bash
# 编译
go build -o bin/api cmd/api/main.go

# 运行
./bin/api        # Linux/Mac
bin\api.exe     # Windows
```

### 5. 验证运行

项目启动后，你会看到类似输出：

```
服务器启动 {"port": 8080}
```

访问健康检查接口：

```bash
curl http://localhost:8080/health
```

或者浏览器访问：`http://localhost:8080/health`

应该返回：

```json
{ "status": "ok" }
```

## 常见问题

### 1. 配置文件找不到

**错误**：`加载配置失败: open configs/config.yaml: no such file or directory`

**解决**：确保在项目根目录运行，或者修改 `cmd/api/main.go` 中的配置文件路径。

### 2. 数据库连接失败

**错误**：`连接数据库失败: dial tcp 127.0.0.1:3306: connect: connection refused`

**解决**：

- 检查 MySQL 服务是否启动
- 检查配置文件中的数据库连接信息是否正确
- 检查防火墙设置

### 3. 端口被占用

**错误**：`bind: address already in use`

**解决**：

- 修改 `configs/config.yaml` 中的端口号
- 或者关闭占用 8080 端口的程序

### 4. 数据库表不存在

项目启动时会自动创建表（通过 GORM AutoMigrate），如果失败：

- 检查数据库用户是否有 CREATE TABLE 权限
- 检查数据库是否存在

## API 测试示例

### 1. 用户注册

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "test@example.com",
    "phone": "13800138000",
    "password": "123456",
    "nickname": "测试用户"
  }'
```

### 2. 用户登录

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "123456"
  }'
```

### 3. 获取个人信息（需要 Token）

```bash
curl -X GET http://localhost:8080/api/v1/user/profile \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

## 停止服务

按 `Ctrl + C` 停止服务，程序会优雅关闭。

## 开发建议

1. **使用热重载工具**（可选）：

   ```bash
   # 安装 air
   go install github.com/cosmtrek/air@latest

   # 运行（会自动监听文件变化并重启）
   air
   ```

2. **查看日志**：日志会输出到控制台，生产环境建议配置日志文件。

3. **数据库迁移**：项目启动时会自动迁移表结构，无需手动执行 SQL。
