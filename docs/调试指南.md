# Go 项目调试指南

## 调试方式

Go 支持多种调试方式，最常用的是使用 **Delve** 调试器。

---

## 方式 1：VS Code 调试（推荐）

### 1. 安装 Delve 调试器

```bash
go install github.com/go-delve/delve/cmd/dlv@latest
```

### 2. 安装 VS Code Go 扩展

在 VS Code 中安装：

- **Go** (由 Go Team at Google 提供)

### 3. 创建调试配置

在项目根目录创建 `.vscode/launch.json`：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Launch API Server",
      "type": "go",
      "request": "launch",
      "mode": "auto",
      "program": "${workspaceFolder}/cmd/api",
      "env": {},
      "args": [],
      "showLog": true,
      "trace": "verbose"
    },
    {
      "name": "Attach to Process",
      "type": "go",
      "request": "attach",
      "mode": "local",
      "processId": 0
    }
  ]
}
```

### 4. 设置断点

1. 在代码行号左侧点击，设置断点（红色圆点）
2. 按 `F5` 启动调试
3. 程序会在断点处暂停

### 5. 调试快捷键

- `F5` - 启动/继续调试
- `F9` - 切换断点
- `F10` - 单步跳过（Step Over）
- `F11` - 单步进入（Step Into）
- `Shift+F11` - 单步跳出（Step Out）
- `Shift+F5` - 停止调试

## 方式 2：GoLand 调试（JetBrains IDE）

### 1. 创建运行配置

1. 点击 `Run` → `Edit Configurations...`
2. 点击 `+` → `Go Build`
3. 配置：
   - **Name**: `API Server`
   - **Run kind**: `Package`
   - **Package path**: `github.com/SwordDragonLee/go-web/cmd/api`
   - **Working directory**: `$ProjectFileDir$`

### 2. 设置断点

- 在代码行号左侧点击设置断点
- 点击 `Debug` 按钮（或 `Shift+F9`）启动调试

### 3. 调试面板

- **Variables**: 查看变量值
- **Watches**: 监视表达式
- **Call Stack**: 调用栈
- **Breakpoints**: 断点管理

## 方式 3：命令行调试（Delve）

### 1. 安装 Delve

```bash
go install github.com/go-delve/delve/cmd/dlv@latest
```

### 2. 启动调试

```bash
# 方式1: 直接调试
dlv debug ./cmd/api

# 方式2: 附加到运行中的进程
dlv attach <进程ID>
```

### 3. Delve 常用命令

```
(dlv) break main.main          # 在 main 函数设置断点
(dlv) break internal/service/user_service.go:48  # 在指定文件行号设置断点
(dlv) continue                 # 继续执行
(dlv) next                     # 下一行（不进入函数）
(dlv) step                     # 下一行（进入函数）
(dlv) print variable           # 打印变量值
(dlv) locals                   # 显示局部变量
(dlv) args                     # 显示函数参数
(dlv) stack                    # 显示调用栈
(dlv) list                     # 显示当前代码
(dlv) exit                     # 退出调试
```

## 方式 4：使用日志调试

### 1. 添加调试日志

```go
import (
    "go.uber.org/zap"
    "github.com/SwordDragonLee/go-web/pkg/logger"
)

func (h *UserHandler) Login(c *gin.Context) {
    logger.GetLogger().Debug("收到登录请求",
        zap.String("username", req.Username))

    // ... 业务逻辑

    logger.GetLogger().Info("登录成功",
        zap.Uint("user_id", user.ID),
        zap.String("username", user.Username))
}
```

### 2. 设置日志级别

在 `configs/config.yaml` 中：

```yaml
server:
  mode: debug # debug 模式会输出更多日志
```

## 调试技巧

### 1. 条件断点

在 VS Code 中：

- 右键点击断点
- 选择 "Edit Breakpoint"
- 输入条件，例如：`userID == 1`

### 2. 日志断点

在断点处添加日志输出，不暂停执行：

- 右键点击断点
- 选择 "Edit Breakpoint"
- 勾选 "Logpoint"
- 输入日志消息：`用户ID: {userID}`

### 3. 监视表达式

在调试时，可以添加监视：

- 变量名：`user.Username`
- 表达式：`len(req.Password) > 6`
- 函数调用：`utils.HashPassword("test")`

### 4. 调用栈查看

查看函数调用链：

- VS Code: 左侧 "Call Stack" 面板
- GoLand: "Debug" 窗口的 "Frames" 标签

## 常见调试场景

### 场景 1：调试登录接口

**设置断点位置**：

```go
// internal/api/handler/user_handler.go
func (h *UserHandler) Login(c *gin.Context) {
    var req dto.LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // 断点1: 检查参数绑定错误
        c.JSON(http.StatusBadRequest, dto.Error(...))
        return
    }

    // 断点2: 检查请求参数
    ip := c.ClientIP()
    userAgent := c.GetHeader("User-Agent")

    // 断点3: 检查登录结果
    loginResp, err := h.userService.Login(&req, ip, userAgent)
    if err != nil {
        // 断点4: 检查登录错误
        c.JSON(http.StatusBadRequest, dto.Error(...))
        return
    }

    c.JSON(http.StatusOK, dto.Success(loginResp))
}
```

**调试步骤**：

1. 在断点 2 设置断点
2. 启动调试（F5）
3. 前端发送登录请求
4. 查看 `req` 变量的值
5. 单步执行，查看 `loginResp` 的值

### 场景 2：调试 Service 层业务逻辑

```go
// internal/service/user_service.go
func (s *userService) Login(req *dto.LoginRequest, ip, userAgent string) (*dto.LoginResponse, error) {
    // 断点1: 检查请求参数
    var user *model.User
    var err error

    // 断点2: 检查用户查找结果
    if user, err = s.userRepo.GetByUsername(req.Username); err != nil {
        // ...
    }

    // 断点3: 检查密码验证
    if !utils.CheckPassword(req.Password, user.Password) {
        // ...
    }

    // 断点4: 检查 Token 生成
    token, err := utils.GenerateToken(user.ID, user.Username, string(user.Role))

    return &dto.LoginResponse{...}, nil
}
```

### 场景 3：调试数据库查询

```go
// internal/repository/user_repository.go
func (r *userRepository) GetByUsername(username string) (*model.User, error) {
    var user model.User
    // 断点: 检查查询结果
    err := r.db.Where("username = ?", username).First(&user).Error
    if err != nil {
        return nil, err
    }
    return &user, nil
}
```

## 调试最佳实践

### 1. 使用日志 + 断点结合

```go
func (h *UserHandler) Login(c *gin.Context) {
    logger.GetLogger().Debug("开始处理登录请求")

    var req dto.LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        logger.GetLogger().Error("参数绑定失败", zap.Error(err))
        // 设置断点，查看 err 详情
        c.JSON(http.StatusBadRequest, dto.Error(...))
        return
    }

    logger.GetLogger().Debug("参数绑定成功", zap.String("username", req.Username))
    // 设置断点，检查 req 的值
}
```

### 2. 使用条件断点

```go
// 只在特定条件下暂停
// 条件: userID == 1
if user.ID == 1 {
    // 断点
}
```

### 3. 使用断言

```go
import "github.com/stretchr/testify/assert"

func (s *userService) Login(...) {
    // 调试时添加断言
    assert.NotNil(user, "用户不应该为 nil")
    assert.NotEmpty(token, "Token 不应该为空")
}
```

### 4. 使用 defer 追踪函数执行

```go
func (h *UserHandler) Login(c *gin.Context) {
    defer func() {
        logger.GetLogger().Debug("Login 函数执行完成")
    }()

    // ... 业务逻辑
}
```

## 常见问题

### Q1: 断点不生效？

**A**: 检查：

1. 是否在 `main` 包中启动调试
2. 代码是否已保存
3. 是否使用了优化编译（`-N -l` 禁用优化）

### Q2: 无法查看变量值？

**A**:

- 确保使用 `debug` 模式编译
- 检查变量是否在作用域内
- 使用 `print` 命令打印变量

### Q3: 调试时程序很慢？

**A**:

- 减少断点数量
- 使用条件断点
- 使用日志代替部分断点

### Q4: 如何调试并发代码？

**A**:

- 使用 `goroutine` 命令查看所有 goroutine
- 使用 `goroutine <id>` 切换 goroutine
- 在关键位置添加日志
